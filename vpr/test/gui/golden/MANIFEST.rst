Golden Images ‚Äî Visual Regression Baseline
===========================================

This directory stores the reference (golden) PNG images for Layer 5 visual
regression tests.  Each image is **2048 √ó 2048 px, 8-bit RGBA**, produced
by VPR's ``save_graphics`` command under ``QT_QPA_PLATFORM=offscreen``.

**Status:** Populated.

Layout
------

The case list lives in ``vpr/test/gui/visual_cases.sh`` (sourced by both
``generate_goldens.sh`` and ``run_visual_regression.sh``): two VPR
invocations sweep the graphics-state variables, one per stage-barrier
group.

* **Pass 1** (``--pack --place --route``) ‚Äî saves at ``placement_done``
  and ``routing_done`` barriers; covers ``set_nets``, ``set_cpd``, and
  ``set_draw_block_internals``.
* **Pass 2** (``--pack --place --route``) ‚Äî saves at ``routing_initial``
  (first routing-stage screen update, mid-flight ``route_ctx``); covers
  ``set_congestion``.

Names are functional (``placement_critical_path_delays``), not numeric
(``cpd=2``). Every name listed below must be emitted by exactly one
``save_graphics`` line in ``visual_cases.sh``.

Regeneration Command
--------------------

.. code-block:: bash

   ./vpr/test/gui/generate_goldens.sh \
       ./build/vpr/vpr \
       ./vtr_flow/arch/timing \
       ./vtr_flow/benchmarks/microbenchmarks

Architecture: ``k6_N10_40nm.xml``, seed: ``1``, display: ``--disp off``.

.. warning::

   Regenerating goldens invalidates all previous SSIM comparisons.
   Only regenerate when an intentional visual change is made (e.g.,
   colour-scheme update, layout algorithm change) and commit the new
   PNGs alongside an entry in the defect log explaining why.
